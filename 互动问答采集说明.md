# 互动问答采集说明

## 文件

- 采集脚本: `tools/fetch_hudong_qa.py`
- 自选股来源: `自选股20260624.txt`
- 输出目录格式: `股票代码_股票名称_互动问答`
- 每只股票目录内包含:
  - `data.json`: 去重后的原始结构化数据，供后续分析使用
  - `00_问答汇总.md`: 每只股票唯一的问答笔记，按答复日期倒序展开一问一答

## 命令触发端点

统一入口都是：

```powershell
python tools\fetch_hudong_qa.py --mode <模式> [其他参数]
```

### 1. 获取全部自选股问答

```powershell
python tools\fetch_hudong_qa.py --mode all-full --max-pages 100
```

### 2. 获取指定股票全量问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-full --code 688456 --max-pages 100
```

### 3. 获取指定股票最近 1 周问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-range --code 688456 --period 1w --max-pages 20
```

### 4. 获取指定股票最近 1 月问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-range --code 688456 --period 1m --max-pages 50
```

### 5. 获取全部自选股增量问答

```powershell
python tools\fetch_hudong_qa.py --mode all-incremental --max-pages 20
```

### 6. 获取指定股票增量问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-incremental --code 688456 --max-pages 20
```

### 7. 获取指定股票最近 1 周内的增量问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-incremental --code 688456 --period 1w --max-pages 20
```

### 8. 获取指定股票最近 1 月内的增量问答

```powershell
python tools\fetch_hudong_qa.py --mode stock-incremental --code 688456 --period 1m --max-pages 50
```

### 9. 只试跑前 5 只自选股

```powershell
python tools\fetch_hudong_qa.py --mode all-incremental --limit-stocks 5 --max-pages 10
```

说明：`stock-range` 会自动使用覆盖模式，只保留本次时间窗口内的结果；`incremental` 模式会与已有 `data.json` 合并，适合长期维护知识库。
## 参数说明

- `--mode`: 命令触发模式，可选 `all-full`、`stock-full`、`stock-range`、`all-incremental`、`stock-incremental`。
- `--period`: 时间窗口别名，支持 `1w`、`1m`、`7d`、`30d`，也支持 `近1周`、`近1月`。
- `--code`: 指定股票代码，多个代码用英文逗号分隔，如 `688456,300034`。
- `--days`: 限定本次抓取最近 N 天内答复的问答；如需输出中也只保留这个时间窗口，请同时使用 `--replace`。
- `--since`: 指定起始答复时间，如 `2026-06-01`。
- `--full`: 全量采集到 `--max-pages`，不按已有数据停止。
- `--incremental`: 读取该股票目录下的 `data.json`，从最新答复时间之后增量更新。
- `--replace`: 不合并旧 `data.json`，只保留本次抓取结果，适合“最近 7 天”这类窗口查询。
- `--max-pages`: 最多翻页数，首次全量可适当调大。
- `--min-delay` / `--max-delay`: 请求间隔随机范围，默认较温和；大批量采集时建议不要调得太低。
- `--include-bse`: 包含北交所代码。
- `--include-funds`: 包含 ETF/基金类代码。

## 说明

脚本默认跳过 ETF/基金与北交所代码，主要覆盖上交所、深交所股票。采集源使用公开互动问答聚合接口，并带随机 User-Agent、随机延迟、重试、去重合并，适合定期放入 Windows 任务计划程序运行。
